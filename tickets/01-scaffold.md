# Ticket 01 — Repo scaffold

## Goal
Stand up a clean Next.js App Router + TypeScript + Tailwind project at the repo
root, with `.gitignore` and README stubs in place, so subsequent tickets have
a working baseline to add code to.

## In scope
- `create-next-app` with App Router, TypeScript, Tailwind, ESLint, **no `src/`**.
- `.gitignore` entries for `.data/`, `.env*.local`, standard Next artifacts.
- `.env.example` with just the `BILLING_PROVIDER` line so it's discoverable
  (full env matrix comes in Ticket 02).
- README placeholder — a single line pointing at `ARCHITECTURE.md` and
  `tickets/`. Full README lands in Ticket 12.
- Baseline commit.

## Out of scope
- Any provider code.
- Any pages beyond the default `create-next-app` home page.
- Any lib code (webhook log, provider resolver) — those are Ticket 02.

## Files touched
- `package.json`, `tsconfig.json`, `next.config.*`, `tailwind.config.*`,
  `postcss.config.*`, `app/layout.tsx`, `app/page.tsx`, `app/globals.css`
  (all generated by `create-next-app`).
- `.gitignore` (extend).
- `.env.example` (create with one line).
- `README.md` (stub).

## Steps
1. Run `npx create-next-app@latest .` — App Router, TS, Tailwind, ESLint, no
   `src/`, no import alias change from default `@/*`.
2. Add to `.gitignore`:
   ```
   .data/
   .env*.local
   ```
3. Create `.env.example` with:
   ```
   BILLING_PROVIDER=stripe
   ```
4. Overwrite `README.md` with a one-liner pointing to `ARCHITECTURE.md` and
   `tickets/README.md`.
5. `npm run build` to confirm the scaffold compiles.
6. Commit: `chore: scaffold Next.js app`.

## Acceptance criteria
- `npm run dev` boots without errors and serves the default page at `/`.
- `npm run build` succeeds.
- `.data/` and `.env.local` would be ignored by git (verified via `git check-ignore`).
- No provider dependencies added yet (no `stripe`, `chargebee`, `@paddle/*`).

## Dependencies
None.
